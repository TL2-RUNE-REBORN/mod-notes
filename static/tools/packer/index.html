.MOD 打包器网页版

Pack in the Browser 零 上 传 · 零 安 装

选一个 mod 文件夹,浏览器就地把 DAT / LAYOUT 编译成引擎二进制格式并封装成 .MOD——与桌面版打包器同一套 Rust 代码编译成 WebAssembly,输出一致。文件全程留在你的电脑里,不经过任何服务器。

说明与边界 Notes & Limits

词缀色阶 —— ■ 传奇橙 = 关键保证 · ■ 套装绿 = 正常状态 · ■ 焰红 = 边界警示
  • 本地打包全程在浏览器的 WebAssembly 沙箱内完成,任何文件都不会上传;页面资源加载完后断网也能打包。
  • 一致BINDAT / BINLAYOUT / RAW 编译与 .MOD 容器封装与桌面版 tl2-mikuro-mod-packer 同一套代码,内容逐字节一致。manifest 中的文件时间戳按浏览器可见的毫秒精度写入(桌面版为 100ns 精度),该字段的微小差异不影响游戏加载与 MOD 激活。
  • 继承附带全量 Runic base UNITS 模板(7731 个文件,gzip 760 KB),BASEFILE 单位继承链在浏览器内完整解析——装备 / 单位类 mod 的编译结果与桌面版一致。
  • 边界整棵 mod 目录会读入内存;数百 MB 的巨型整合组件建议直接用桌面版打包器。
MPP 关卡寻路数据不生成(等价于桌面版 --mpp none)。含关卡 LAYOUT 的 mod(新地图 / 改地图)请用桌面版打包——MPP 烘焙需要 base 游戏的关卡几何,网页版只附带了 UNITS 模板。纯数值 / 单位 / 物品 / 技能类 mod 不受影响。
Ⅲ

内置自检 Self-test

用内置的「天煞星宠物 MOD」样例验证你当前浏览器里的整条打包链:输出须与桌面版参照逐字节一致(SHA-256),并验证文件时间戳注入链路。
